\chapter{Introduction and objectives}

\section{Vision}
The goal is an agent capable of completing a Pok\'{e}mon \textbf{Nuzlocke}-style challenge using deep reinforcement learning. The commercial target is Pok\'{e}mon Brilliant Diamond (BDSP): beat mandatory trainers without losing the run.

\section{Nuzlocke rules (problem framing)}
\begin{itemize}
  \item Only the first Pok\'{e}mon encountered in each new area may be caught.
  \item Fainted Pok\'{e}mon are lost for the remainder of the run.
  \item If the player blacks out (no usable Pok\'{e}mon), the run ends and must restart.
\end{itemize}
These rules create \textbf{irreversible} outcomes and \textbf{limited resources}, i.e.\ sequential decision-making under risk.

\section{Success metrics (project-level)}
Among others: gauntlet completion rate; depth reached before failure; remaining resources after a successful run; generalization to other gauntlets or orderings; and model size needed for competence.

\section{Scope}
\begin{itemize}
  \item \textbf{In scope:} battle decision-making, simulator integration, training and evaluation of RL policies.
  \item \textbf{Out of scope:} full game world traversal, overworld exploration, and non-battle mechanics.
\end{itemize}
